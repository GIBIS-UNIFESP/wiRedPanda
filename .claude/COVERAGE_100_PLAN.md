**Recurring exclusion patterns to recognize on sight** (don't re-derive from scratch each time):
1. A function's closing `}` shown as uncovered when the function itself clearly ran → compiler-generated cleanup for a local non-trivial-destructor object's exception-unwind path that's never taken. Exclude the closing brace line.
2. A `qCDebug`/`qCWarning` call split across multiple physical lines shows the first line covered but continuation lines `#####` even when the log call is definitely reached → the streamed arguments only evaluate when the category is enabled (default verbosity is 0 in tests), and gcov only credits the *first* line for the always-evaluated enabled-check. Fix: reformat onto one physical line (matches this codebase's existing single-line convention anyway) rather than fiddling with `Comment::setVerbosity()` in tests.
3. `= default` special member functions (esp. virtual destructors) may show as uncovered even when heavily exercised, since the trivial body gets inlined at each call site with no attributable standalone execution. Exclude with a one-line justification rather than chasing it.
4. Before writing a test for an `if (someString.isEmpty())` fallback fed by a helper function, check every return path of that helper first — if it has no path that returns an empty string (common for "resolve to a name or a sensible default" helpers), the fallback is dead and no circuit will ever trigger it.
5. A variable name built as `prefix + counter + ...` can never exactly equal a bare reserved/keyword string — don't spend time hunting a label that would trigger an `isReserved(fullVarName)`-style check if the counter is unconditionally part of that name.
6. When constructing a "run out of a shared resource" test (pins, IDs, etc.) with a MIX of element types, scene iteration order (`QGraphicsScene::items()`, roughly most-recently-added-first) determines which element's own overflow check actually fires — order assumptions are easy to get backwards. The robust fix: make every element competing for the resource the *same type*, so whichever one is starved when the resource runs out throws from the identical source line regardless of order.
7. A `GraphicElement`'s label is *never* genuinely empty once it round-trips through save+load as an IC boundary port — `ICLoader::buildPortLabels()` falls back to the class name (e.g. `"InputSwitch"`) whenever the label is blank, even an explicit `setLabel("")`. Don't assume "leave it unlabeled" reaches an `isEmpty()`/`"_unnamed"` fallback; verify with a `qWarning()` probe first (see the IC dedup test in `TestSystemVerilogCodeGenUnit`) before writing the test or excluding the line.
8. A multi-line statement built from chained calls (`.arg().arg()...`, not just streamed `qCDebug`/`qCWarning`) can show its *first* line as 0-hits while the continuation lines show real counts — the opposite direction from pattern 2, but the same root cause (gcov attributing the statement's "entry" oddly across lines). Fix: reformat onto one physical line; don't assume the surrounding loop is unreached just because the first line reads `#####`.
9. To build a *real* multi-level IC fixture for a unit test (not an integration `.panda` file), construct the sub-circuit directly (`WorkSpace`/`CircuitBuilder`), call `workspace.save(path)`, then `IC::loadFile(path, dir)` — this round-trips through the actual production save/load path (real port-label derivation, real boundary-Node proxying) without needing to hand-craft a `.panda` blob. This is the reusable "minimal IC" helper technique deferred during `ArduinoCodeGen.cpp` and used to close out `SystemVerilogCodeGen.cpp`'s IC-reserved-word/dedup gap.
10. Before assuming an out-of-topological-order element reference is dead/impossible, check whether the surrounding code has an explicit recursive fallback for it (e.g. `otherPortNameImpl()` recursing into `generateLogicExpressionImpl()` when an upstream gate hasn't been assigned yet) — if so, it's live defensive-but-real code for when `Scene`/caller-supplied element order isn't strictly topological, and deserves a real test (build the elements vector directly, order it deliberately), not an exclusion.
